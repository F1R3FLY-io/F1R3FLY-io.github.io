## üö¶ Getting Started by Role

### For Developers
1. **Core Platform**: Start with [`f1r3fly`](https://github.com/F1R3FLY-io/f1r3fly) for main infrastructure
2. **Smart Contracts**: Use [`rholang-rs`](https://github.com/F1R3FLY-io/rholang-rs) and [`rholang-language-server`](https://github.com/F1R3FLY-io/rholang-language-server)
3. **Frontend Development**: Check [`embers-frontend`](https://github.com/F1R3FLY-io/embers-frontend) for web applications
4. **Editor Support**: Use [`rholang-nvim`](https://github.com/F1R3FLY-io/rholang-nvim) or [`rholang-vscode-client`](https://github.com/F1R3FLY-io/rholang-vscode-client)

### For Infrastructure Engineers  
1. **Deployment**: Use [`F1r3bu1ld3r`](https://github.com/F1R3FLY-io/F1r3bu1ld3r) for visualization
2. **Cloud Infrastructure**: Check [`IaC`](https://github.com/F1R3FLY-io/IaC) for Python automation
3. **Monitoring**: Explore [`Sankey_block_explorer`](https://github.com/F1R3FLY-io/Sankey_block_explorer) for data visualization

### For Researchers
1. **Language Design**: Explore [`MeTTa`](https://github.com/F1R3FLY-io/MeTTa) for type system research
2. **Parser Technology**: Use [`bnfc`](https://github.com/F1R3FLY-io/bnfc) for grammar tools
3. **File Systems**: Investigate [`F1R3FLYFS`](https://github.com/F1R3FLY-io/F1R3FLYFS) for distributed storage

### For Users
1. **Wallets**: Use [`embers`](https://github.com/F1R3FLY-io/embers) for asset management
2. **Social Platform**: Try [`f1r3sky`](https://github.com/F1R3FLY-io/f1r3sky) for decentralized social
3. **Code Editing**: Use [`lightning-bug`](https://github.com/F1R3FLY-io/lightning-bug) for browser-based development

## üèó Technology Stack Overview

### Languages
- **Scala**: Core platform and type systems (f1r3fly, MeTTa)
- **Rust**: Performance-critical components and tooling (rholang-rs, embers, clients)
- **TypeScript/JavaScript**: Frontend applications and backends (f1r3sky ecosystem, embers-frontend)
- **Clojure**: Development tools (lightning-bug)
- **Python**: Infrastructure automation (IaC)
- **Swift**: Mobile and system extensions (f1r3drive-extension)
- **Java**: File system research (F1R3FLYFS)
- **Lua**: Editor plugins (rholang-nvim)

### Infrastructure
- **Distributed Systems**: High-throughput transaction processing
- **Language Tools**: Comprehensive Rholang development ecosystem
- **Cloud Infrastructure**: Python-based deployment automation
- **Development Tools**: Multi-editor support and visualization tools

## ü§ù Contributing

Each repository has its own contribution guidelines. Generally:

1. **Document First** - Follow F1R3FLY.io's documentation-first methodology
2. **Use Project Template** - New repositories should follow our [standard template](https://github.com/F1R3FLY-io/F1R3FLY-io.github.io/blob/main/docs/Project_Template.md)
3. **Follow Language Conventions** - Respect each project's language-specific practices
4. **Security First** - Follow security best practices for distributed systems
